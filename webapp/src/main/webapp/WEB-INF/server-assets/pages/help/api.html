REST API (beta version)

Biosamples exposes an API for external usage.
If you have any problem or suggestion please contact: Rui Pereira <rpereira@ebi.ac.uk>;

This is an on-going work, so some parts are not yet implemented (security, error codes catalog, etc.), but it's already possible to have access to:

  • Information about a specific Group Accession;
  • Information about a specific Sample Accession;
  • Search Groups with paging support;
  • Search Samples inside a specific Group with paging support;

You can consult the descriptions of each of those bellow, and you can always follow the "More details" link to know all the details about it.
For each service the follow information will be available:

  • Security;
  • Parameters;
  • Invocation example;
  • Response example;
  • Error codes;
  • XSD - Xml Schema definition;

Available Services

  1. Group Accession ( http://$SERVER/biosamples/xml/group/$GROUPACCESSION)
    • It returns all information about a SampleGroup;
    • If you specify the sampleslist argument with true value, it will also return a list with the Samples Accessions;
    • E.g.  /biosamples/xml/group/SAMEG82620;
    • More details;
  1. Sample Accession ( http://$SERVER/biosamples/xml/sample/$SAMPLEACCESSION)
    • It returns all information about a Sample;
    • E.g.  /biosamples/xml/sample/SAME546131;
    • More details;
  1. Search Sample Groups ( http://$SERVER/biosamples/xml/group/query=$QUERY)
    • It returns all the groups that match the query (e.g.  /biosamples/xml/group/query=cancer);
    • It is possible to use paging and sorting (e.g.  /biosamples/xml/group/query=cancer&sortby=relevance&sortorder=descending&page=2&pagesize=50);
    • Always return a result summary information; 
      <ResultQuery>
<SummaryInfo>
<Total>7766</Total>
<From>51</From>
<To>100</To>
<PageNumber>2</PageNumber>
<PageSize>50</PageSize>
</SummaryInfo>
<BioSampleGroup id="SAMEG72185"/>
<BioSampleGroup id="SAMEG126084"/>
...
</ResultQuery>
    • More details;
  1. Search Samples inside a Sample Group ( http://$SERVER/biosamples/xml/groupsamples/$SAMPLEGROUP/query=$QUERY)
    • It returns all the samples that are inside the SampleGroup parameter and also match the query(e.g.  http://www.ebi.ac.uk/biosamples/xml/groupsamples/SAMEG23004/query=cancer);
    • It is possible to use paging (e.g.  e.g. /biosamples/xml/groupsamples/SAMEG23004/query=cancer&page=3&pagesize=10);
    • Always return a result summary information; 
      <ResultQuery>
<SummaryInfo>
<Total>3316</Total> 
<From>21</From>
<To>40</To>
<PageNumber>2</PageNumber>
<PageSize>20</PageSize>
</SummaryInfo>
<BioSample id="SAMEA477370"/>
<BioSample id="SAMEA507158"/>
...
</ResultQuery>
    • More details;
